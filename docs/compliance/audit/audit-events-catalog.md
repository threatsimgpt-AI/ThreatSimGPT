**Overview**

This catalog defines all audit events generated by ThreatSimGPT systems. Each event type includes its definition, required fields, severity level, and compliance mapping.

**Event Schema: authentication.login.success**

```json
{
  "event_type": "authentication.login.success",
  "required_fields": {
    "actor.user_id": "string",
    "actor.ip_address": "string",
    "context.session_id": "string"
  },
  "optional_fields": {
    "actor.user_agent": "string",
    "context.auth_method": "enum[password, sso, api_key, mfa]",
    "context.mfa_used": "boolean",
    "context.mfa_method": "enum[totp, sms, email, hardware]"
  }
}
```

**10. Event Type Naming Convention**

Events follow a hierarchical naming convention:

```
{category}.{subcategory}.{action}[.{modifier}]
```

| Component | Description | Examples |
|-----------|-------------|----------|
| category | High-level domain | authentication, authorization, data |
| subcategory | Specific area | login, role, user |
| action | What happened | success, failure, created |
| modifier | Optional qualifier | admin, bulk, forced |
