**1. Definitions**

| Term | Definition |
|------|------------|
| **Personal Data** | Any information relating to an identified or identifiable natural person |
| **Processing** | Any operation performed on Personal Data |
| **Data Subject** | An identified or identifiable natural person |
| **Sub-processor** | Any third party engaged by Processor to process Personal Data |
| **Data Protection Laws** | GDPR and applicable national implementing legislation |
| **Services** | The ThreatSimGPT services described in the main agreement |

---

**2. Scope and Purpose**

**Section 2.1 Subject Matter**

This DPA governs the processing of Personal Data by Processor on behalf of Controller in connection with the provision of the Services.

**Section 2.2 Duration**

This DPA shall remain in effect for the duration of the Processor's processing of Personal Data on behalf of the Controller.

**Section 2.3 Nature and Purpose of Processing**

| Element | Description |
|---------|-------------|
| **Nature** | Cloud-based threat simulation platform |
| **Purpose** | Enable Controller to conduct security threat simulations |
| **Processing Operations** | Storage, computation, transmission, deletion |

**Section 2.4 Categories of Personal Data**

The Processor may process the following categories of Personal Data:

| Category | Examples |
|----------|----------|
| Account Data | Email address, username, display name |
| Authentication Data | Hashed passwords, session tokens |
| Usage Data | API calls, timestamps, IP addresses |
| Simulation Content | User-provided scenario parameters |
| Support Data | Ticket content, communications |

**Section 2.5 Categories of Data Subjects**

- Controller's employees and authorized users
- Individuals whose data may be included in simulation content (at Controller's discretion)

---

**3. Controller Obligations**

The Controller shall:

**Section 3.1 Lawfulness**
Ensure that its instructions for processing comply with Data Protection Laws.

**Section 3.2 Data Subject Information**
Provide appropriate notices to Data Subjects regarding the processing.

**Section 3.3 Legal Basis**
Ensure a valid legal basis exists for the processing.

**Section 3.4 Data Accuracy**
Ensure Personal Data provided to the Processor is accurate and up-to-date.

**Section 3.5 Data Minimization**
Ensure only necessary Personal Data is submitted to the Services.

**Section 3.6 Instructions**
Provide documented instructions for processing Personal Data.

---

**4. Processor Obligations**

The Processor shall:

**Section 4.1 Documented Instructions**

Process Personal Data only on documented instructions from the Controller, including regarding:
- Transfers to third countries
- Unless required by EU or Member State law

If the Processor believes an instruction infringes Data Protection Laws, it shall immediately inform the Controller.

**Section 4.2 Confidentiality**

Ensure that persons authorized to process Personal Data:
- Have committed to confidentiality or are under statutory obligation
- Process Personal Data only as instructed

**Section 4.3 Security Measures**

Implement appropriate technical and organizational measures, including:

| Measure | Implementation |
|---------|----------------|
| **Pseudonymization** | User IDs for internal processing |
| **Encryption** | AES-256 at rest, TLS 1.3 in transit |
| **Confidentiality** | Access controls, authentication |
| **Integrity** | Input validation, audit logs |
| **Availability** | Redundancy, backups, disaster recovery |
| **Resilience** | High availability architecture |
| **Testing** | Regular security testing, vulnerability scanning |

**Section 4.4 Sub-processing**

**Section 4.4.1 Authorization**
The Controller provides general authorization for the Processor to engage Sub-processors, subject to:
- Processor maintaining a list of Sub-processors
- Processor informing Controller of changes with 30 days' notice
- Controller's right to object to new Sub-processors

**Section 4.4.2 Sub-processor List**

Current Sub-processors:

| Sub-processor | Service | Location | Safeguards |
|---------------|---------|----------|------------|
| [Cloud Provider] | Infrastructure hosting | EU | DPA, ISO 27001 |
| [Email Provider] | Transactional email | EU | DPA |
| [Support Platform] | Customer support | EU | DPA |

**Section 4.4.3 Sub-processor Contracts**
The Processor shall ensure Sub-processor contracts impose equivalent data protection obligations.

**Section 4.5 Data Subject Rights**

The Processor shall assist the Controller in responding to Data Subject requests by:
- Providing technical measures to enable request fulfillment
- Promptly notifying Controller of requests received directly
- Not responding to requests without Controller authorization (unless legally required)

**Section 4.6 Security and Breach Notification**

**Section 4.6.1 Assistance**
The Processor shall assist the Controller in ensuring compliance with security obligations, breach notification, and DPIAs.

**Section 4.6.2 Breach Notification**
The Processor shall notify the Controller without undue delay (and within 48 hours) upon becoming aware of a Personal Data breach, including:
- Nature of the breach
- Categories and approximate number of Data Subjects
- Categories and approximate number of records
- Likely consequences
- Measures taken or proposed

**Section 4.7 Deletion and Return**

Upon termination or Controller request, the Processor shall:
- Return all Personal Data to the Controller, and/or
- Delete all Personal Data (unless storage is required by law)
- Certify deletion upon request

The Processor shall complete deletion within 30 days of termination.

**Section 4.8 Audit Rights**

The Processor shall:
- Make available all information necessary to demonstrate compliance
- Allow for and contribute to audits and inspections
- Immediately inform Controller if an instruction infringes Data Protection Laws

Audit conditions:
- Reasonable advance notice (minimum 30 days, except for urgent matters)
- During normal business hours
- At Controller's expense
- Subject to confidentiality obligations
- Third-party audit reports (e.g., SOC 2) may satisfy audit requirements

---

**5. International Transfers**

**Section 5.1 Transfer Restrictions**

The Processor shall not transfer Personal Data outside the EU/EEA unless:
- Controller has provided prior written authorization, and
- Appropriate safeguards are in place per Chapter V of GDPR

**Section 5.2 Transfer Mechanisms**

Where transfers are authorized, the following mechanisms apply:

| Destination | Mechanism |
|-------------|-----------|
| UK | Adequacy decision |
| Countries with adequacy | Adequacy decision |
| US | Standard Contractual Clauses + supplementary measures |
| Other | Standard Contractual Clauses |

**Section 5.3 Standard Contractual Clauses**

Where required, the Parties agree to incorporate the EU Standard Contractual Clauses (Commission Implementing Decision (EU) 2021/914), specifically:
- Module Two: Controller to Processor
- With the following options selected:
  - Clause 7: Docking clause included
  - Clause 9(a): General written authorization for sub-processors
  - Clause 11: Independent dispute resolution (optional)
  - Clause 17: Governing law of [Member State]
  - Clause 18: Courts of [Member State]

---

**6. Liability**

**Section 6.1 Allocation**
Each Party shall be liable for damages caused by its breach of this DPA or Data Protection Laws.

**Section 6.2 Indemnification**
Each Party shall indemnify the other against claims, damages, and expenses arising from its breach of this DPA.

**Section 6.3 Limitations**
Liability limitations in the main agreement apply to this DPA, except liability cannot be limited for:
- Breaches of confidentiality obligations
- Indemnification obligations
- Violations of Data Protection Laws

---

**7. Term and Termination**

**Section 7.1 Term**
This DPA commences on the effective date of the main agreement and continues until termination of the Services.

**Section 7.2 Survival**
Obligations regarding confidentiality, data deletion, and audit rights survive termination.

**Section 7.3 Termination for Breach**
Either Party may terminate this DPA if the other Party materially breaches and fails to cure within 30 days of notice.

---

**8. General Provisions**

**Section 8.1 Conflicts**
In case of conflict between this DPA and the main agreement, this DPA prevails regarding Personal Data processing.

**Section 8.2 Amendments**
This DPA may only be amended in writing signed by both Parties.

**Section 8.3 Severability**
If any provision is unenforceable, remaining provisions continue in effect.

**Section 8.4 Governing Law**
This DPA is governed by the laws of [Jurisdiction].

**Section 8.5 Entire Agreement**
This DPA, together with the main agreement and any annexes, constitutes the entire agreement regarding Personal Data processing.
